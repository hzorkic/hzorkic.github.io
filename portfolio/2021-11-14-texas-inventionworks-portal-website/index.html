Texas Inventionworks Portal Website

The Problem

During the summer of 2020, my team and I build a website to automate and digitize the operational workflow of Texas Inventionworks (The University of Texas at Austin’s Engineering Scool’s Makerspace). As UT Austin shut down all non-essential campus operations during the “lockdown,” we were tasked with creating an application that allowed us to keep track of all of the students that came into the space and all of the transactions they made.

If you want to learn more about Texas Inventionworks, check out the website here !

We need to develop a tool to…

  1. Track who is coming into the space and for how long for contract tracing.
  2. Track what tools each person checks out for contract tracing.
  3. Create a way to minimize staff-student interaction to avoid the spread of COVID-19.

The Research

This project was unique from a user research perspective as the end users of the application were staff members who also use the space like students. Furthermore, we were able to constantly iterate and ideate solutions to the problem with each other through the product development.

Brainstorm and Conceptualize

The project needed to contain a staff facing application and a student facing application- kind of like a portal. Connecting these two pieces, we needed to create a backend database to store out data and a server to interact with the database.

Create a Prototype

After many failures, consulting with friends, and self research, we decided on the following tech stack:

  1. AWS AppSync for the server (-less) server
  2. GraphQL for the API
  3. DynamoDB - the NoSQL, non-relational database
  4. AWS Amplify go host our site
  5. React for the frontend design

Bonus:

  1. Figma for multi-purpose design planning
  2. Agile Scrum group software development

Product Analysis

After a few re-builds, we were able to deploy our website to the public right as classes began for the new semester, Our organization was able to open up to the student body and act as the first facility available for undergraduate students to do research and projects in.

The website was designed and built in just under 3 months, so there were obviously a few bugs. My team continued to fix bugs ad-hoc throughout the semester.

Improve

After out first product was developed and tinkered with, we brought a new member onto the team- an experienced software developer. Over the course of the semester, he converted all of our home-spun components (features such as navigation bars, warning pop-ups, information bars, etc.) into Material UI components, but kept the rest of the website the same!

I am currently working to optimize the application for data collection and analysis by integrating data pulled from applications that students frequently use and embedding login quizzes that ask why a user has entered the space. This will allow us to collect data such as what classes use the space most, what students use the space most, what areas and machines are used most in the space, etc. Stay tuned for another blogpost about that!
